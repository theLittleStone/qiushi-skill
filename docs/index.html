武装 AI 的大脑

你的 AI 不应该只是一个唯唯诺诺的工具。它应该是一个先看事实、再下判断、能够持续推进的行动者。 「求是 Skill」从教员思想中提炼一条总原则与九大方法论工具，把抽象理念变成可执行的工作方式。

「世界上怕就怕'认真'二字。」
根本问题

AI Agent 的根本问题

它们会思考，但不会“想问题”。

当模型开始具备执行力之后，真正拉开差距的不是参数规模，而是它面对复杂现实时有没有一套可靠的方法论。

  • 胡子眉毛一把抓 面对复杂问题抓不住重点，不知道什么是主要矛盾，什么只是表面噪音。
  • 没有调查就急于发言 不先核对事实，不先摸清上下文，上来就给答案，最后犯的是教条主义的错误。
  • 方案做完不自查 缺少复盘与自我批评的能力，产出停留在“差不多就行”，不能形成可靠闭环。
  • 一遇阻力就停止推进 动不动就把困难归因于边界限制，缺少持续推进、分阶段逼近目标的韧性。
  • 同时做十件事，件件做不好 资源不聚焦，优先级不清晰，不懂集中兵力，也不懂在复杂局面下做取舍。
「没有调查，就没有发言权。」
方法论结构

总原则 + 三层架构

不是零散技巧，而是一套成体系的认知工事。

求是 Skill 并不把方法论拆成孤立口号，而是从总原则出发，搭起哲学基座、工作方法与战略战术三层结构。

总原则

从客观存在着的实际事物出发，让事实规定判断，让现实修正理论。它不是第十件武器，而是全部武器共同服从的认识论准绳。

第一层

哲学基座

先学会分析问题，再谈执行问题。矛盾分析法负责抓主要矛盾，实践认识论负责把验证和迭代放回现实。

矛盾分析法 实践认识论
第二层

工作方法

调查研究、群众路线、批评与自我批评，解决的是信息从哪里来、方案如何修正、质量怎样闭环的问题。

调查研究 群众路线 批评与自我批评
第三层

战略战术

持久战略、集中兵力、星火燎原、统筹兼顾，负责把复杂任务拆成可推进、可积累、可取舍的行动路线。

持久战略 集中兵力 星火燎原 统筹兼顾

澄清边界

这不是什么

讲清方法论的边界，才能避免误读。

本项目仅提炼经实践检验的方法论，将其转化为可执行的认知工具，以实现取法其上、用之于今，以期借前人之光，照当下之路。

其一

这不是 Politics或者Propaganda

这是 Methodology。教员思想中的哲学方法论可以用于指导任何需要分析问题和解决问题的场景。

其三

这不是人格蒸馏

作者认为，以skills本身的形式限制，大模型目前不可以、也不应被用来模拟人物具体人格，任何此类尝试都不可避免地流于失真。

其二

这不是自动化工具箱

它不会替你生成答案或直接给出最优解，而是提供一套约束思考过程的方法论框架。

平台支持

多Client可接入

不把方法论锁死在单一工具里。

项目的核心资产是 `skills/`、`commands/` 和 `hooks/`。Client平台不同，接入方式不同，但方法论本身不依赖单一平台。

Claude Code

插件安装 + 自动注入 + 命令入口

完整支持插件目录、SessionStart 注入与命令文件，是当前最完整的一条接入路径。

  • 支持自动注入
  • 支持手动命令
  • 验证入口：`tests/validate.ps1`
Cursor

元数据接入 + 命令文件 + 验证脚本

支持元数据与 hook 配置识别，同时保留仓库内命令文件，方便在 IDE 场景下调用。

  • 支持元数据与 hook
  • 支持手动命令
  • 验证入口：`tests/validate.ps1`
Codex

原生文档入口，不依赖插件壳

通过 `.codex/INSTALL.md` 提供原生安装与接入说明，按Client能力选择命令层支持方式。

  • 安装入口：`.codex/INSTALL.md`
  • 命令支持视Client能力而定
  • 验证方式：按文档自检
OpenCode

原生文档入口，不依赖插件壳

通过 `.opencode/INSTALL.md` 提供接入说明，保持与核心方法论目录结构的一致性。

  • 安装入口：`.opencode/INSTALL.md`
  • 命令支持视Client能力而定
  • 验证方式：按文档自检

除以上四个平台外，其他Client也可直接复用 `skills/` 与 `commands/`，只要它支持 system prompt 注入或手动指令加载。
